================================================================================
GRADIENT PATTERNS
================================================================================

1. HERO GLOW (Ambient Background Effect)
   Five overlapping radial gradients with semantic colors, blurred 50px.
   Works identically in both themes (colors already theme-adjusted).

2. HERO TITLE (Spectral Text Gradient)
   Linear 135° through all 5 semantic colors, animating via background-position.
   Creates mesmerizing flowing effect without flashing.

3. CARD GRADIENT (Subtle Glass Effect)
   135° linear overlay with 2% indigo opacity over solid background.
   Gives depth without obscuring content.

4. TEXT GRADIENT (Headlines)
   135° linear from indigo to purple to indigo.
   Applied with background-clip: text.

5. BORDER GRADIENT (Dividers)
   Horizontal gradient from indigo to purple.
   Draws the eye without being jarring.

6. MESH GRADIENT (Section Backgrounds)
   Two overlapping radial gradients (indigo + green) at 20% and 80% positions.
   Opacity 2-4% for subtle ambient effect.

================================================================================
DESIGN PHILOSOPHY
================================================================================

Goal 1: SEMANTIC CLARITY
─────────────────────────
Four method statuses must be instantly recognizable:
- Green (#10B981 dark, #059669 light) = "owns" = "this runs"
- Amber (#F59E0B dark, #D97706 light) = "overrides" = "you win"
- Purple (#8B5CF6 dark, #7C3AED light) = "overridden" = "wins here, not below"
- Red (#EF4444 dark, #DC2626 light) = "shadowed" = "dead code"

Developers must know method status within 200ms of cursor landing.

Goal 2: PREMIUM AESTHETIC
──────────────────────────
- Sophisticated color layering (5 background levels)
- Smooth gradient flows (hero, titles, dividers)
- Elegant shadow depths (4 levels)
- Zero jarring transitions between themes
- Feels like a $500+ app, not a free tool

Goal 3: ACCESSIBILITY
──────────────────────
- WCAG AA on all text/critical UI (4.5:1 minimum)
- Most content AAA (7:1+)
- No color-only information (always pair with icons/text)
- Tested with color-blind simulators
- Screen-reader friendly structure

QUICK REFERENCE
================================================================================

When in doubt, use:
- Page background: var(--bg-0)
- Card background: var(--bg-1)
- Body text: var(--text-primary)
- Labels/hints: var(--text-secondary)
- Primary button: var(--indigo-50) on var(--bg-0)
- Success/owned: var(--green-50)
- Warning/overrides: var(--amber-50)
- Error/shadowed: var(--red-50)
- Info/overridden: var(--purple-50)

All variables defined in css_variables_complete.css
